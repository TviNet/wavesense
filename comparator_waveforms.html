Comparator Design Mental Model - Waveforms

CP1: Equality vs Inequality (Mode 0)

Feature: Match mode 0 (equality) - both MatchEq and Match assert on exact equality.
Key Transitions:
• me00 and m00 move together with equality
• Both signals assert when t0 == c00 (0xDEADBEEF)

CP2: Inequality Inversion (Mode 8)

Feature: Match mode 8 (inequality) - m00 is logical NOT of equality while me00 remains equality.
Key Transitions:
• me00 stays as equality detector
• m00 inverts: asserts when values are NOT equal

CP3: NAPOT-Masked Compare (Mode 1)

Feature: Mode 1 uses a mask derived from tdata2 (first 0-bit from LSB). With t0=0xFFFF_FFFF_FFFF_FF00, masked compare holds across variations in the low byte.
Key Transitions:
• m00 stays 1 despite unequal me00
• Masking is effective - low byte variations ignored

CP4: Relational Comparators (Modes 2 & 3)

Feature: Modes 2 and 3 implement unsigned >= and < operations. At the threshold (equal values), mode 2 asserts, mode 3 deasserts.
Key Transitions:
• Mode 2 (>=): 0x100 >= 0xFF (0), 0x100 >= 0x100 (1), 0x100 >= 0x200 (0)
• Mode 3 (<): 0x100 < 0xFF (0), 0x100 < 0x100 (0), 0x100 < 0x200 (1)

CP5: Masked Low/High-Half Compares (Modes 4 & 5)

Feature:
• Mode 4: (csr_low & mask_low) == tdata2_low, with mask_low = tdata2_high
• Mode 5: (csr_high & mask_high) == tdata2_low, with mask_high = tdata2_high
Key Transitions:
• Mode 4: Only bits within mask_low contribute
• Mode 5: Compare targets the masked CSR high half against tdata2_low
