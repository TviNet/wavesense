CP1: Equality vs Inequality Mode 0

Feature: MatchEq always performs equality, Match follows match_mode (0: eq, 8: neq)
me00 m00 match_mode t0 c00 0 0 0 0xDEADBEEF 0xFEEDFACE 1 1 0 0xDEADBEEF 0xDEADBEEF 0 0 0 0xDEADBEEF 0xDEADBEF0 1 1 0 0xDEADBEEF 0xDEADBEEF
Observation: Look for m00 and me00 moving together with equality

CP2: Inequality Inversion Mode 8

Feature: Mode 8 inequality - m00 becomes logical NOT of equality while me00 remains equality
me00 m00 match_mode t0 c00 1 0 8 0x12345678000000AA 0x12345678000000AA 0 1 8 0x12345678000000AA 0x12345678000000AB 1 0 8 0x12345678000000AA 0x12345678000000AA
Observation: m00 is logical NOT of equality while me00 remains equality

CP3: NAPOT-Masked Compare Mode 1

Feature: Uses mask derived from first 0-bit in tdata2 scanning from LSB to MSB
me00 m00 match_mode t0 c00 0 1 1 0xFFFFFFFFFFFFFF00 0xAAAAAAAAAAAAAA55 0 1 1 0xFFFFFFFFFFFFFF00 0xAAAAAAAAAAAAAA80 0 1 1 0xFFFFFFFFFFFFFF00 0xAAAAAAAAAAAAAA00
Observation: m00 stays 1 despite unequal me00, indicating masking is effective

CP4: Relational Comparators Modes 2 & 3

Feature: Unsigned >= and < between csr_values and tdata2
me00 m00 match_mode t0 c00 0 0 2 0x100 0xFF # 0x100 >= 0xFF? NO 1 1 2 0x100 0x100 # 0x100 >= 0x100? YES 0 1 2 0x100 0x200 # 0x100 >= 0x200? NO 0 1 3 0x100 0xFF # 0x100 < 0xFF? NO 1 0 3 0x100 0x100 # 0x100 < 0x100? NO 0 0 3 0x100 0x200 # 0x100 < 0x200? YES
Observation: At the threshold (equal), mode 2 asserts, mode 3 deasserts

CP5: Masked Low/High-Half Compares Modes 4 & 5

Feature: Mode 4: (csr_low & mask_low) == tdata2_low | Mode 5: (csr_high & mask_high) == tdata2_low
me00 m00 match_mode t0 c00 0 1 4 0xFF000000AA00 0x55000000AA00 0 0 4 0xFF000000AA00 0x55000000AB00 0 1 5 0xF0F00000D0A00000 0xD0ADBEEF00000000 0 0 5 0xF0F00000D0A00000 0xC0ADBEEF00000000 0 1 5 0xF0F00000D0A00000 0xD0A0123400000000
Observation: Mode 4: Only bits within mask_low contribute. Mode 5: Compare targets masked CSR high half
